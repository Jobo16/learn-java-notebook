- 包装类型的核心作用：**让基本数据类型拥有对象的特性，填补基本数据类型与 Java 面向对象设计之间的鸿沟**。
- 4 个核心需求场景：
    
    - 支持泛型（泛型不支持基本数据类型）。
    - 支持 `null` 值（表示 “无值”、“未知值”，避免歧义）。
    - 提供丰富的工具方法（数据转换、进制转换等）。
    - 符合面向对象编程特性（参与继承、多态等操作）。
    
- 辅助机制：自动装箱 / 拆箱（Java 5+），让基本类型和包装类型的转换更简洁。
- 权衡：基本数据类型效率高、占用空间小，适合简单运算；包装类型功能丰富，适合复杂场景（泛型、数据库交互等）。
