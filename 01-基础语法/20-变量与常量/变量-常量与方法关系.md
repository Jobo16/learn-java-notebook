一、定义层面的根本区别

变量表示“可以变化的数据”。  
常量表示“一旦确定就永远不变的数据”。

在 Java 中，常量通常通过 static final 来定义。

例如：

int age = 18;

这是变量，可以在后续代码中被重新赋值。

static final int MAX_AGE = 120;

这是常量，一旦赋值完成，就不能再被修改。

final 负责“不能再改值”。  
static 负责“属于类，而不是属于某个对象”。

二、是否允许修改，这是最直接的判断标准

变量可以多次赋值：

age = 20;  
age = 25;

常量只能赋值一次：

static final int MAX = 10;  
// MAX = 20; 编译期直接报错

只要你看到 final 修饰的字段，第一反应就是它的值在整个生命周期内保持不变。

三、是否依赖对象，这是 static 的关键作用

普通成员变量属于对象。  
每 new 一个对象，就有一份独立的变量。

例如：

class Person {  
int age;  
}

Person p1 = new Person();  
Person p2 = new Person();

p1.age 和 p2.age 是两块不同的内存。

static 成员属于类。  
整个 JVM 中只有一份。

例如：

class Config {  
static int timeout;  
}

无论 new 多少个 Config，对应的 timeout 都是同一个。

static final 组合在一起，表示“类级别、全局唯一、不可修改”的值。

四、内存和生命周期上的区别

变量的生命周期通常和对象绑定。  
对象被回收，变量随之消失。

static final 常量在类加载时初始化。  
只要类没有被卸载，它就一直存在。

对于基本类型和 String 常量，编译期常量还可能被直接内联到字节码中，提高性能。

五、命名和代码风格上的明显差异

变量命名遵循小驼峰：

count  
userName

常量命名使用全大写加下划线：

MAX_COUNT  
DEFAULT_TIMEOUT

这不是语法要求，是约定俗成的强信号。  
看到全大写字段，几乎可以直接判断它是常量。

六、什么时候用变量，什么时候用常量

用变量的典型场景。  
对象的状态会变化。  
同一类对象之间的数据不同。  
运行过程中需要被重新赋值。

用 static final 常量的典型场景。  
配置值、固定阈值、数学常数。  
枚举意义的标识值。  
业务规则中的固定约束。

例如：

static final double PI = 3.1415926;  
static final int ORDER_PAID = 1;

七、一个快速判断口诀

能不能改。  
是不是每个对象一份。  
是不是整个系统统一不变。

只要答案是“不能改”“类级别唯一”“全局统一”，就应该使用 static final。
